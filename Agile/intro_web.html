Refresh of Web Programming

Web Programming consists of the technologies for creating web pages and is the basis of creating web-based applications.

Languages

The technologies and languages come in a few different categories and these languages must work together for the web application to be successful.

  • Visual and Markup
    • (X)HTML
    • CSS
    • XML
  • Client Side Languages
    • JavaScript
    • AJAX (mixture of Javascript and XML)
  • Server Side Languages
    • PHP/Perl/Python
    • ASP.Net
    • Java EE
    • Many more

What does an HTTP request and response look like?

Every interaction with a web-based application comes in the form of an HTTP request. These requests are accepted by a web server and a response is generated with the contents of the HTML/XML for the client. The following is a sample request and response for a request to www.google.com

HTTP Request

GET / HTTP/1.1
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.9.0.6) Gecko/2009011913 Firefox/3.0.6
Host: www.google.com
Accept: */*

HTTP Response

HTTP/1.1 200 OK
Cache-Control: private, max-age=0
Date: Tue, 17 Feb 2009 05:29:21 GMT
Expires: -1
Content-Type: text/html; charset=UTF-8
Set-Cookie: PREF=ID=049714b659131a8c:TM=1234848561:LM=1234848561:S=YVk4UjhXBSerd2pT; expires=Thu, 17-Feb-2011 05:29:21 GMT; path=/; domain=.google.com
Server: gws
Transfer-Encoding: chunked

(The sample request was generated from www.httpviewer.net)

How to handle state in a web-based application?

State is held in every type of application when there is user interaction. The only problem is that the HTTP protocol is a stateless protocol, which means that it is impossible to maintain state in the protocol. There are a few ways of maintaining state in a web application such as using cookies, sessions, and postbacks.

Cookies

Cookies are text files that are stored on the client's computer. These files hold the variables that are required for state to be maintained. One downside to this method of maintaining state is that users can disable cookies as a security measure. Cookies have been used in the past as a way of tracking what web pages a user visits.

Sessions

Sessions are similar to cookies, but the information is stored on the server. Users are identified by a unique identifier which comes from their browser and the session information could be stored in a few places on the web server. These sessions can be stored in files on the webserver, in a database, or in the web server's memory. Sessions are better than cookies because they are stored on the server side and cannot be tampered with by malicious software and users.

Post Backs

Sessions and cookies are used to maintain state on the server and client machines, respectively. The last option is where all of the state information is passed between requests. The state information is placed in a form that is submitted whenever a user tries to go to another web page. This method has some benefits when cookies and sessions cannot be utilized.
